Welcome to Page Timestamp Finder

Track when web pages were published or last updated with this simple, powerful extension.

Why We Built This Extension

Don't you hate it when you're on a webpage and you're unsure if the information is new or old?

Sadly, few websites clearly display when their content was published or last updated. This can be frustrating when you need to know if you're looking at current information.

Page Timestamp Finder solves this problem by finding both the published and last modified dates of a page. Simply click the extension icon to find out, giving you confidence in the content you're reading.

For the best experience, we recommend pinning this extension to your toolbar for easy access.
1
Pin Page Timestamp Finder to your toolbar

Click the pin icon next to our extension to keep it visible at all times.

Pinning gives you quick access to the extension's features from any webpage.
2
Click the extension icon

To find the timestamp for a page, just click the extension icon. The extension finds both "published" and "modified" dates.

When timestamps are available you'll see the details in an overlay in the right corner.
3
Customize your experience

You can set your preferred date and time format. Right-click the extension icon, select "Options", and choose the format that works best for you.

How the Extension Works

We intelligently scan the webpage for both published and modified timestamps, checking various locations where a date might be hiding. Our approach follows a logical sequence, prioritizing the most reliable sources first:

1

Schema.org Structured Data

First, we analyze any Schema.org structured data (in JSON-LD or microdata format) embedded in the page. This often contains the most accurate datePublished (or dateCreated as a fallback) and dateModified properties directly from the author. Content is securely sanitized to handle malformed control characters.

2

Meta Tags

Next, we look for direct indicators in the page's meta tags. This includes standard tags like last-modified as well as Open Graph (OG) tags like og:updated_time (for modified) and og:published_time (for published).

3

HTML Content Patterns

If the above checks don't yield a result, we start looking at the visible content of the page. We search for HTML5 <time> elements and common phrases or CSS classes (like .post-date or .updated) that often indicate dates.

4

URL Patterns

If a publication date still hasn't been found, we scan the website's address (URL) for common date patterns (like /2023/10/15/) often used by blogs and news sites.

5

Unix Timestamp Parsing

Supports scanning structured elements and page content for both 10-digit (seconds) and 13-digit (milliseconds) Unix timestamps.

6

HTTP Last-Modified Header

As a final fallback for the modified date, we examine the Last-Modified header sent by the web server. While this can sometimes be less precise, it can still provide a useful indication.

Note: We deliberately ignore some generic server headers and the system's "current time" to prevent incorrect publication dates. We also reject dates that appear to be in the future.
